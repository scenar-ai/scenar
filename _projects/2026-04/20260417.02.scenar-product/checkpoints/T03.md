## What was built

Extracted the generic scenario playback engine from the Stigmer repo into two standalone TypeScript packages in the Scenar monorepo:

### @scenar/core (pure TypeScript, zero dependencies)
- **10 source modules** across 7 domains: scenario types, narration types, viewport transform, timing constants, timeline computation, DOM scroll utilities, cursor position math, data-attribute contract
- **6 test files, 27 tests** — all pure-function unit tests
- **Public API** via barrel export with full TSDoc coverage

### @scenar/react (React + Framer Motion)
- **ScenarioPlayer** decomposed from 681 LOC monolith into 8 files:
  - `useStepProgression.ts` — step advancement (timer + time-source paths)
  - `usePlaybackProgress.ts` — RAF-driven progress bar
  - `ScenarioPoster.tsx` — play overlay
  - `ScenarioControls.tsx` — transport bar
  - `SpeedMenu.tsx` — speed popover
  - `ScenarioPlayer.tsx` — slim orchestrator (~210 LOC)
- **useStepInteractions** decomposed from 997 LOC monolith into 15 files:
  - 8 per-action effect files in `effects/`
  - `dom-helpers.ts` — shared DOM resolution
  - `context.ts` — `InteractionContext` interface
  - `warnings.ts` — dev-mode timing warnings
  - `useBrowserStepInteractions.ts` — setTimeout path
  - `useTimeSourceStepInteractions.ts` — frame-driven path
  - `useStepInteractions.ts` — thin dispatcher (~50 LOC)
- **Straight moves**: Cursor, ViewportTransformLayer, TimeSource, VideoExportContext, useNarrationManifest, useNarrationPlayback, PlaybackCoordinator
- **DemoViewport** decoupled from Stigmer `tokens.ts` — all layout constants are now optional props with engine defaults

## Design decisions

1. **Layered packages**: `@scenar/core` has zero React dependency. Someone building a non-React renderer can use the types and timing math without pulling in React + Framer Motion.

2. **Engine stays TypeScript-native**: `ScenarioStep<T>` with generic data payload. Proto mapping will happen in `@scenar/sdk` (T05) via `createScenario()`. This keeps the engine free of protobuf runtime dependencies.

3. **DemoViewport is configurable, not opinionated**: Canonical width, min zoom, shell height, and wrapper classes are all optional props. Stigmer passes its token values in T06; third-party consumers get sensible defaults.

4. **Cursor icons use CSS custom properties**: Replaced Stigmer's `hsl(var(--foreground))` / `hsl(var(--background))` with `currentColor` and `var(--scenar-cursor-stroke, white)`. Host apps can override cursor appearance without importing a theme.

5. **useNarrationManifest URL is configurable**: The second parameter `resolveManifestUrl` overrides the default `/demos/{id}/manifest.json` convention. Stigmer passes its convention; other hosts use their own.

6. **Conditional hooks in useStepInteractions**: The dispatcher calls either `useBrowserStepInteractions` or `useTimeSourceStepInteractions` based on context. This is a conditional hook call — the condition (timeSource presence) is stable for the lifetime of a component, so it doesn't violate Rules of Hooks in practice. ESLint suppressions are documented.
